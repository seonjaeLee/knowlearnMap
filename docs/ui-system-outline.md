# UI 체계 정리 개요 (작업 가이드)

기능 추가보다 **UI/UX 통일**을 우선할 때, **어디에 무엇을 정의하는지** 헷갈리지 않도록 하는 목차입니다.  
내용은 진행하면서 채워 넣고, `docs/ui-history.md`에 반영합니다.

## 0. 작업 순서 (권장)

1. **공통 레이아웃·쉘** — `MainLayout`, 노트북 3열, 탭 영역 (아래 §1)
2. **토큰·타이포 뎁스** — `index.css` + 표준 제목 단계 (§2)
3. **폼 상태 매트릭스** — 표로 고정 후 코드 적용 (§3)
4. **팝업(모달) 공통 구도** — 헤더·본문·푸터 (§4)
5. **버튼 계층** — primary / secondary / … / disabled (§5)
6. **MUI vs 순수 CSS 경계** — 정책 문장 + 화면별 이관 (§6)

한 번에 전부 고치지 않고, **만지는 화면·컴포넌트부터** 위 순서에 맞춰 치환합니다.

## 1. 공통 레이아웃·쉘

### 1.1 포함 범위

- [ ] 전역: `MainLayout` — GNB/LNB/본문 스크롤
- [ ] 노트북: 좌 문서 / 중 탭·콘텐츠 / 우 스튜디오
- [ ] 탭 바: 채팅·지식그래프·사전·외부 링크 버튼 정렬

### 1.2 정리 시 적을 것

- 사용 중인 **루트 클래스명** (예: `.panel-center`, `.tab-content`)
- **최소 폭·스크롤** 규칙 (어느 영역에 가로 스크롤이 생기는지)
- **패널 접힘** 시 동작 요약

## 2. 타이포·타이틀 뎁스

### 2.1 제목 단계 (예시 — 확정 시 숫자·크기만 수정)

| 단계 | 용도 | 크기·굵기 (토큰/값) |
|------|------|---------------------|
| H1 | 페이지 제목 | `—` |
| H2 | 섹션 / 카드 묶음 | `—` |
| H3 | 서브섹션 / 모달 제목 | `—` |
| 본문 | 기본 텍스트 | `--font-size-base` 등 |
| 보조 | 캡션·메타 | `--color-text-secondary` 등 |

### 2.2 확인할 파일 (점진적)

- [ ] `index.css` `:root`에 `--title-*` 또는 기존 `--font-*` 매핑

## 3. 폼 요소 — 상태 매트릭스

**코드를 바꾸기 전에** 아래 표를 채우면, 토큰·컴포넌트 적용 순서가 명확해집니다.

### 3.1 단일 입력 (`input` / `textarea` / `select`)

| 상태 | 배경 | 테두리 | 글자 | placeholder | 커서 |
|------|------|--------|------|---------------|------|
| 기본(쓰기) | | | | | |
| 읽기 전용 | | | | — | |
| 비활성 | `var(--form-control-disabled-bg)` 등 | | | `var(--form-control-disabled-placeholder)` | `not-allowed` |
| 포커스 | | 강조색 | | | |

### 3.2 복합 폼 (라벨 + 입력 행)

| 패턴 | 설명 | 비고 |
|------|------|------|
| 쓰기만 | | |
| 읽기만 | | |
| 혼합 | 일부 필드만 수정 | |

### 3.3 MUI 사용 화면

- **Theme overrides** 또는 공통 `sx` 패턴으로 위 표와 **동일한 색·두께**가 나오도록 매핑한다고 적기.

## 4. 팝업(모달) 공통 구도

### 4.1 구조

```
[ 오버레이 ]
  [ 다이얼로그 ]
    [ 헤더 — 제목 · 닫기 ]
    [ 본문 — 스크롤 영역 ]
    [ 푸터 — 주요 · 보조 버튼 ]
```

### 4.2 정리 시 적을 것

- 패딩·최대 높이·`border-radius`
- 푸터 버튼 정렬 (우측 정렬 / gap)
- **신규 모달**부터 이 구도 적용, 기존 모달은 수정 시 하나씩

---

## 5. 버튼 계층

| 역할 | 용도 | 배경·글자 (토큰) | 비고 |
|------|------|------------------|------|
| Primary | 주 액션 | `--color-accent` | |
| Secondary | 보조 | | |
| Ghost / Text | 덜 강조 | | |
| Danger | 삭제 등 | | |
| Disabled | | `--btn-disabled-bg` 등 | |

- 순수 `<button>` + 클래스 vs MUI `Button` variant — §6과 함께 결정.

### 6.1 정책 (문장으로 고정)

### 6.2 이관 시

- 해당 페이지를 건드릴 때만, 위 §2~§5에 맞춰 조정.
